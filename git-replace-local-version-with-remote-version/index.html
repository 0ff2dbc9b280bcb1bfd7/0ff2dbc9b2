Git Replace Local Version With Remote Version

Oct 29th, 2016 12:04 pm

git pull is the primary command while acquiring the remote changes, under the hood this is same as running git fetch and git merge but this won’t help when you are sure to replace the local content with remote while conflicts.

Using the below set of commands to hard reset local version with remote:

git fetch origin

git reset –hard origin/master

Explanation:

  • git fetch – This will imports commits from a remote repository into your local repo.
  • git reset –hard origin/master – This will reset Reset current HEAD to the specified state, Hard is to ignore any changes in your local copy.

Tip:

Make a alias git-replace: **alias git-replace=”git fetch origin && git reset –hard origin/master”**
