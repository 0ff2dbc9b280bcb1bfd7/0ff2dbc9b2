Difference Between Service Oriented Architecture (SOA) and Micro Services

Oct 29th, 2016 9:51 am

Service-oriented architecture (SOA): Is an architectural pattern in computer software design in which application components provide services to other components via a communications protocol, typically over a network.

Microservices: Is a software architecture style in which complex applications are composed of small, independent processes communicating with each other using language-agnostic APIs

The core difference between SOA and microservices lies in the size and scope. As the word “micro” suggests, it has to be significantly smaller than what SOA tends to be. Microservice is a small(er) independently deployable unit. Beware of very small microservice antipattern – nanoservice. A SOA can be either a monolith or it can be comprised of multiple microservices.

Example: Say you have a SOA component which is written in Java to authencateUsers, and has 5 end points, like auth..user, auth..adminuser, auth..googleuser, etc. And consumer expects a session id from this service. But internally the code which is prodiving you a token can be a seperate microservice, which may be written in any programming language unlike the parent SOA using Java, can be independently called by other applications, etc.

Enable Htaccess in Apache

Oct 29th, 2016 8:45 am

By default htaccess is disabled in Apache, to enable htaccess follow the below steps:

  1. Enable Apache rewrite module – sudo a2enmod rewrite
  2. Restart Apache – sudo service apache2 restart

If this didn’t work, follow the below steps:

  1. Edit apache2.conf file – sudo vim /etc/apache2/apache2.conf
  2. Locate correct Directory tag
  3. Change AllowOverride None to AllowOverride All
  4. Change Require all denied to Require all granted
  5. Restart Apache – sudo service apache2 restart
