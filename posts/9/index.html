Implementing Scheduled / Delayed Queue Using ActiveMQ

Oct 29th, 2016 10:37 am

ActiveMQ has in-built support for scheduler, this adds support to deliver a message to the listener after x time. To enable this support add chedulerSupport=”true” in broker tag in activemq.xml, broker tag with scheduler support looks like this:

<broker xmlns=“http://activemq.apache.org/schema/core&#8221; brokerName=“localhost” dataDirectory=“${activemq.data}” schedulerSupport=“true”>

To schedule a ActiveMQ message any scheduled property need to be set using setLongProperty). Avaliable properties are:

  • AMQ_SCHEDULED_DELAY
  • AMQ_SCHEDULED_PERIOD
  • AMQ_SCHEDULED_REPEAT
  • AMQ_SCHEDULED_CRON

Open New Terminal Tab Using Command in Linux

Oct 29th, 2016 10:08 am

Linux default (gnome in Ubuntu) terminal doesn’t provide an API / Command to open a new tab, however new window can be opened using gnome-terminal command. Terminal provides a shortcut Ctrl+Shift+t to open a new tab, leveraging this we can use xdotool to emulate the new tab shortcut.

xdotool according to Linux.org:

The xdotool is a utility used from the terminal or in a script to manually perform keyboard input. The commands can also be used to make a script of many xdotool commands to perform large tasks. Later articles will cover the xdotool ability to perform mouse input as well as window and desktop manipulation.

Install xdotool:

# Ubuntu

sudo apt-get install xdotool

# Fedora / CentOS:

sudo yum install xdotool

Now trigger the new tab shortcut using: xdotool key ctrl+shift+t

Create a alias for ease: alias newtab=”xdotool key ctrl+shift+t”

Install and Enable PHP Mcrypt in Linux

Quick cheat sheet to install PHP Mcrypt in Ubuntu:

sudo apt-get install php5-mcrypt

sudo php5enmod mcrypt

sudo service apache2 reload

Cent / RedHat equivalent:

yum install php-mcrypt*

Alternatively you can compilie PHP from source with the –with-mcrypt[=DIR] parameter to enable mcrypt.

Get Git Commit Log by Author

Oct 29th, 2016 9:58 am

Git log’s –author flag can be used to list the commits made by a author, working example:

git log –author=”joe@gmail.com”

Git search for author is however not a strict comparision, if the given input exists in author, git prints that out, for example using @example.com in search field will list out all commits of author who uses example.com email:

git log –author=”@example.com”

Tips:

  • Using –no-pager flag prints output to stdout instead of pager (new window)
  • Using -1 will restrict git log to one result. (1 can be replaced with any numeric)

To get the recent commit made by a author use:

git log -1 –author=”joe@gmail.com”

Difference Between Service Oriented Architecture (SOA) and Micro Services

Oct 29th, 2016 9:51 am

Service-oriented architecture (SOA): Is an architectural pattern in computer software design in which application components provide services to other components via a communications protocol, typically over a network.

Microservices: Is a software architecture style in which complex applications are composed of small, independent processes communicating with each other using language-agnostic APIs

The core difference between SOA and microservices lies in the size and scope. As the word “micro” suggests, it has to be significantly smaller than what SOA tends to be. Microservice is a small(er) independently deployable unit. Beware of very small microservice antipattern – nanoservice. A SOA can be either a monolith or it can be comprised of multiple microservices.

Example: Say you have a SOA component which is written in Java to authencateUsers, and has 5 end points, like auth..user, auth..adminuser, auth..googleuser, etc. And consumer expects a session id from this service. But internally the code which is prodiving you a token can be a seperate microservice, which may be written in any programming language unlike the parent SOA using Java, can be independently called by other applications, etc.
