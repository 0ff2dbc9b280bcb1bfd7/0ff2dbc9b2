Copy File From Remote Machine to Local Using SCP in Linux

Nov 6th, 2016 5:38 pm

Scp stands for secure copy program which copies files between hosts on a network. It uses ssh for data transfer, and uses the same authentication and provides the same security as ssh.

copy remote file to local:

scp -r joe@127.20.20.20:/home/joe/file.csv ~/remote/joe/

This will copy remote’s file.csv to the local directory ~/remote/joe

Convert MKV Files to MP4 Using FFMPEG in Linux

Nov 6th, 2016 5:33 pm

ffmpeg is a swiss army knife for video files, here’s how to convert mkv file to mp4:

ffmpeg -i my.mkv -vcodec copy -acodec copy my.mp4

If you have a set of mkv files in a directory, you can convert all of them to mp4 using below command:

for i in *mkv; do ffmpeg -i $i -vcodec copy -acodec copy $i.mp4; done

Step 2: Import:

Once you download the library (I’m using a customized version), you should be having two important things, 1 the highlight.pack.js file and 2 the styles folder. Styles folder have multiple styles (themes) that you can choose from, I’m using github-gist style, which is very similar to the gist.github.com. Redirect a Site Using Apache

Oct 29th, 2016 1:28 pm

Using apache we can easily redirect a site to another site preserving the end url, that means:

Redirecting: bestblog.co/hello-world to www.bestblog.co/hello-world

Add redirect code to config:

Go to Apache enabled sites: cd /etc/apache2/sites-enabled, edit your config like below:

<VirtualHost *>
  ServerName example.com
  Redirect 301 / http://www.example.com/
</VirtualHost>

This will redirect example.com to www.example.com

How to Validate Spring RequestBody

Oct 29th, 2016 1:16 pm

When working to post request’s using Spring framework, we often need validation, consider the below snippet:

@RequestMapping(value = "/user/search", method = RequestMethod.POST, produces = "application/json")
public String search(@RequestBody SearchUserRequest searchUserRequest) {
  // ... some code
}

In this code if you are required to validate SearchUserRequest, you might do this using code like this:

 

if(searchUserRequest.getUserEmail() != null || searchUserRequest.getUserName()!= null ...etc)

But this not what we are looking for if we have classes with quite some number of properties, instead we can make use of JavaX validation annotation @Valid, let’s rewrite the above code with @Valid annotation:

@RequestMapping(value = "/user/search", method = RequestMethod.POST, produces = "application/json")
public String search(@Valid @RequestBody SearchUserRequest searchUserRequest) {
  // ... some code
}

Along with this we are required to mention what all properties in the SearchUserRequest class are required, we do this my using JavaX constraints.

public class SearchUserRequest{
  @NotNull
  private String name;

  @NotNull
  private String email;

  ....
}

This will make sure that request has name and email set. Tip: @NotNull annotation will not consider empty strings, if you want to validate empty strings as well use, @NotBlank annotation from Hibernate validator
